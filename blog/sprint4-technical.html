Solving a simple problem....

We all have challenges from time to time we need to solve one way or another. Sometimes it can even be a really simple problem, like, at the start while setting up this blog I needed to link the CSS file, which carries all the styling of a homepage, to my blog pages. Since the CSS file is in a separate folder to the index (main page) file and the blog, I didn't realise it would cause the issues it did.

See, when you are trying to link your styling page to the index file you need to put in a separate link (like the URL or address to a web page) telling the page where it can find the styling guideline for the page. Because the file is in a separate folder I need to put a ../ in front of the folder I can find the CSS file in, so I should have used the link/address:
"../styles/main.css" - but instead I had just used styles/main.css". "Styles" is the name of the folder the CSS file is in, "main" is the name of the file and .css is the file type.

It was something I should have picked up quickly and a very minor detail I didn't even think of while being so focused on getting my code right. In order to try and solve this problem I ended up spending a lot of google searching and trying new things back and forth. In the end I asked my flatmate if he could see what was going wrong - and surely enough, I just missed a couple of full stops!

Woops! But at least I have now learned to be careful and how to navigate properly between folders while writing code.

All the different ways to problem solve...

There are many ways to problem solve. Some of the problem solving methods we use while coding are:

Pseudocode (simply put outlining what the code do/should be doing through out using comments //)
Trying something
Rubber ducky method (explaining the problem and solution to a rubber ducky, animal, toy, statue - anything other than human!)
Reading error messages
Console.logging
Googling
Asking your peers for help
Asking coaches for help
Improving your process with reflection

Personally I feel most confident so far googling, reading through the error messages, using console.logging, just keep trying a few different things or asking peers/coaches for help if I get really frustrated! Even though I much prefer to solve the problem myself.
While I don't feel uneasy using reflection or pseudocode, I feel this just comes naturally while coding and after.
The only problem solving method I am a bit uneasy about and probably wouldn't be doing is the rubber ducky method, while I can understand how it might have benefitted some, to me I feel a bit uncomfortable doing it in a home where it is not just myself :) instead I will explain to my flatmate to see if I truly understand the problem and solution instead.
